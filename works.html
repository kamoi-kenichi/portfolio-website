LIMITSルーレット

JavaScriptで開発した、デジタルアートバトル「LIMITS」の個人練習向けテーマルーレット＋20分タイマーアプリです。

  • App
  • JavaScript
  • Animation
概要
具体（モノ）と抽象（テーマ）を組み合わせて出題し、20分タイマーを自動開始する練習アプリ
担当
企画 / UIデザイン / コーディング / アニメーション実装
ツール / 言語
Visual Studio Code / HTML / CSS / JavaScript
制作期間
約2日
（設計1日 / 実装1日）

課題と解決

練習時に課題になりやすい「テーマ決めに迷う時間」を減らし、発想から制作開始までをスムーズにできるよう設計しています。 また、テーマ確定から20分タイマー開始までを1画面で完結させることで、 本番に近い緊張感と高揚感を再現できる構成を目指しました。

具体（モノ）と抽象（テーマ）をルーレットで組み合わせ、両方の結果が確定すると、 公式ルールと同じ20分タイマーが自動でスタートする仕様にしています。

デザイン面では「北欧ナチュラル × 犬」をコンセプトに、集中を妨げない配色と余白設計を意識しました。

さらに、START時とTIME UP時には犬が跳び上がってぴょこんとおじぎするアニメーションと効果音を加え、 練習の緊張感の中にも親しみやすさを感じられる演出にしています。

Web App

Java 会員制タスク管理アプリ

Spring Bootを活用して一から構築した、本番環境対応の会員制タスク管理（CRUD）アプリケーションです。

AWS（EC2）上でのサーバー構築、Nginxによるリバースプロキシ、独自ドメインの設定やHTTPS化まで、インフラ全般の整備も自ら行いました。

  • Java (Spring Boot)
  • Spring Security
  • MySQL
  • AWS (EC2) / Nginx
  • Thymeleaf
概要
フロント：HTML / CSS / JavaScript (Thymeleaf)
バック：Java (Spring Boot, Spring Security)
インフラ：AWS EC2, MySQL, Nginx, Let's Encrypt
ツール / 言語
Visual Studio Code / Java / HTML / CSS / MySQL
制作期間
約1週間（アプリ開発5日 / インフラ構築・デプロイ2日）

💡工夫したポイント：鉄壁のセキュリティと本番稼働

Spring Securityを導入し、データベース側でBCryptを用いたパスワードの暗号化を実施しました。
また、本番環境特有のCSRF（クロスサイトリクエストフォージェリ）攻撃を防ぐため、フォームに自動生成トークンを組み込むなど、堅牢な防衛システムを構築しています。

インターネット上に公開するにあたり、Nginxを「受付係」として配置し、通信内容を守るためのHTTPS化（SSL証明書の導入）も行いました。

UIは毎日のタスク管理が楽しくなるよう、ログイン画面から一覧画面まで心地よい「北欧ナチュラル」テイストで統一しています。

サーバーヘルプデスクの実務経験を活かし、開発中の403エラーや404エラーに対して、エラーログから原因を論理的に切り分け、インフラ層からアプリケーション層まで、システム全体を俯瞰したトラブルシューティングを実践しました。

因果の刃

  • Graphic Design
  • Visual Design
概要

ツール / 言語
Photoshop
制作期間
約2時間（コンセプト設計・構図設計・ビジュアル制作）

💡工夫ポイント

ワープ・ゆがみツールを使用し、言葉を刃物状に再構成。
刺さった瞬間を起点にひび割れを広げることで、誹謗中傷が内面の崩壊へと繋がる因果関係を表現しました。

生成結果をワンクリックでコピーし、生成AIへのプロンプトとして利用可能。
工夫点
北欧ナチュラルテイストのUI設計、モダンなJavaScriptによるUI/UXの実装
制作期間
約1週間（ロジック設計、UIデザイン、実装・微調整）

💡工夫したポイント

ただのランダム生成器ではなく、Adobe Firefly等での生成を想定した「結果をコピーする」機能を実装しました。ボタンを押すと状態が変化するアニメーションや、画面上のガーランド・街並みなどのイラストパーツを用いた「アトリエ感」のあるデザインにこだわっています。

また、CSSのブレイクポイント（メディアクエリ）を活用し、スマホとPCそれぞれで最適な「イラストのあしらい」を表示するレスポンシブデザインを実装しました。

UI設計は、私のポートフォリオ全体のテーマである「北欧ナチュラル」な配色と読みやすさを考慮した「M PLUS Rounded 1c」で統一し、心地よい操作体験を追求しました。

概要
疾走感溢れる痛快なロックサウンドに乗せて歌い上げる、笑顔と涙の特別授業。
担当
コンセプト設計 / 作詞 / 楽曲生成（プロンプト調整） / MV制作 / サムネイル制作
ツール
Suno AI / Cubase / Adobe Firefly / Clipchamp
制作期間
約4日 (作詞・作曲・リマスター処理・画像生成)

💡工夫ポイント

  • エレキギターと疾走感のあるバンドサウンドを軸に、別れの悲しさを吹き飛ばすようなパワフルなロックチューンに仕上げ、Cubaseでのリマスタリングでさらに音圧と熱量を高めました。

  • ビジュアル制作（Adobe Firefly）では、六花という幻の存在が消える時が近づいているというシリアスなテーマをあえて「ドヤ顔で熱弁する六花と、それを楽しそうに聞く麦」というコミカルな教室のシーンに変換しました。Key作品らしい「ドタバタで楽しい日常」の空気感を1枚の絵に凝縮しています。

※本楽曲は、Key作品『anemoi』の登場人物・速川六花の物語から深い感動を受け、一人のファンとして心からの感謝と敬意を込めて制作したインスパイア楽曲です。
公式企業・団体様とは一切関係ありません。
